## 1. 项目概述

本系统是一个招聘岗位信息管理平台，旨在实现招聘岗位的全生命周期管理，包括岗位创建、审批、发布、关闭等流程。系统支持多角色权限控制，满足不同用户的业务需求。

---

## 2. 用户角色定义

| 角色 | 角色编码 | 职责描述 | 登录要求 |
|-----|---------|---------|---------|
| **招聘管理员** | RECRUITER | 创建、编辑、删除、批量导入岗位；查看岗位列表与详情；发起审批 | 需登录 |
| **审批人** | APPROVER | 对处于"待审批"状态的岗位进行审批（通过/驳回），可查看岗位列表 | 需登录 |
| **普通访客** | VISITOR | 浏览已发布的岗位信息 | **无需登录** |

## 3. 登录注册需求

### 3.1 登录页面

#### 3.1.1 页面结构
| 元素 | 类型 | 说明 | 约束 |
|-----|------|-----|------|
| 用户名输入框 | input | 用户账号输入 | 必填，最大50字符 |
| 密码输入框 | input[type=password] | 用户密码输入 | 必填，最大100字符 |
| 登录按钮 | button | 提交登录请求 | 点击触发登录 |
| 忘记密码链接 | link | 跳转密码找回（预留） | 可选 |
| 角色选择 | select | 选择登录角色 | 下拉选择（管理员/审批人/访客） |

#### 3.1.2 登录流程
```
用户输入用户名密码 → 选择角色 → 点击登录 → 后台验证 → 返回token和角色信息 → 跳转到对应首页
```

#### 3.1.3 登录API
- **接口地址**: `POST /api/v1/auth/login`
- **请求体**:
```json
{
    "username": "string (必填)",
    "password": "string (必填)",
    "role": "string (RECRUITER/APPROVER/VISITOR, 必填)"
}
```
- **响应体**:
```json
{
    "code": 200,
    "data": {
        "token": "string (JWT令牌)",
        "userId": "number (用户ID)",
        "realName": "string (真实姓名)",
        "role": "string (角色编码)"
    },
    "message": "string"
}
```

### 3.2 注册页面（可选扩展）

#### 3.2.1 页面结构
| 元素 | 类型 | 说明 | 约束 |
|-----|------|-----|------|
| 用户名输入框 | input | 用户账号输入 | 必填，唯一，最大50字符 |
| 密码输入框 | input[type=password] | 用户密码输入 | 必填，最小6位 |
| 确认密码输入框 | input[type=password] | 确认密码 | 需与密码一致 |
| 真实姓名输入框 | input | 用户真实姓名 | 必填，最大50字符 |
| 角色选择 | select | 注册角色 | 管理员可指定，普通用户默认访客 |
| 注册按钮 | button | 提交注册请求 | 点击触发注册 |
| 返回登录链接 | link | 跳转登录页面 | 点击返回 |

#### 3.2.2 注册API
- **接口地址**: `POST /api/v1/auth/register`
- **请求体**:
```json
{
    "username": "string (必填)",
    "password": "string (必填)",
    "realName": "string (必填)",
    "role": "string (RECRUITER/APPROVER/VISITOR, 默认VISITOR)"
}
```

## 4. 前端页面需求（按角色划分）

### 4.1 页面总览

| 页面 | 招聘管理员 | 审批人 | 访客 | 路径 |
|-----|-----------|-------|------|------|
| 首页/岗位列表 | ✓ | ✓ | ✓ | `/` |
| 岗位详情 | ✓ | ✓ | ✓ | `/positions/{id}` |
| 新增岗位 | ✓ | ✗ | ✗ | `/positions/add` |
| 编辑岗位 | ✓ | ✗ | ✗ | `/positions/{id}/edit` |
| Excel上传 | ✓ | ✗ | ✗ | `/positions/upload` |
| 审批页面 | ✗ | ✓ | ✗ | `/positions/{id}/approve` |
| 登录页面 | ✓ | ✓ | - | `/login` |

### 4.2 访客页面

#### 4.2.1 访客首页（无需登录）
- **页面路径**: `/`
- **功能描述**: 展示已发布的招聘岗位列表
- **页面元素**:
  - 页面标题：招聘岗位列表
  - 搜索框：按岗位名称搜索
  - 岗位卡片列表：展示岗位名称、描述、状态（仅显示已发布）
  - 查看详情按钮：跳转岗位详情页

#### 4.2.2 访客岗位详情页
- **页面路径**: `/positions/{id}`
- **功能描述**: 展示单个岗位的详细信息
- **页面元素**:
  - 返回按钮：返回首页
  - 岗位标题
  - 岗位描述
  - 发布时间
  - 状态标签（已发布）

### 4.3 审批人页面

#### 4.3.1 审批人首页
- **页面路径**: `/`
- **功能描述**: 展示待审批岗位列表和所有岗位概览
- **页面元素**:
  - 侧边栏导航
  - 统计卡片：待审批数量、已审批数量
  - 待审批列表：表格展示，包含操作按钮
  - 全部岗位列表：支持筛选和搜索

#### 4.3.2 审批操作页面
- **页面路径**: `/positions/{id}/approve`
- **功能描述**: 对岗位进行审批操作
- **页面元素**:
  - 岗位信息展示区域
  - 审批意见输入框（可选）
  - 通过按钮
  - 驳回按钮
  - 返回按钮

### 4.4 招聘管理员页面

#### 4.4.1 管理员首页
- **页面路径**: `/`
- **功能描述**: 岗位管理总览
- **页面元素**:
  - 侧边栏导航（岗位管理、审批管理、数据统计）
  - 统计卡片：草稿数、待审批数、已发布数、已关闭数
  - 快速操作：新增岗位、批量导入
  - 岗位列表：支持多条件筛选、状态标签

#### 4.4.2 新增/编辑岗位页面
- **页面路径**: `/positions/add` / `/positions/{id}/edit`
- **功能描述**: 创建或编辑岗位信息
- **页面元素**:
  - 表单字段：岗位名称（必填）、岗位描述（可选）
  - 状态显示（编辑时）
  - 提交按钮（保存草稿）
  - 提交审批按钮（草稿→待审批）
  - 取消按钮

#### 4.4.3 Excel上传页面
- **页面路径**: `/positions/upload`
- **功能描述**: 批量导入岗位数据
- **页面元素**:
  - 文件选择器
  - 上传按钮
  - 导入结果展示：成功数量、失败数量、失败原因列表

## 5. 页面流程图

### 5.1 登录流程
```
访问首页 → 未登录 → 跳转登录页 → 登录成功 → 根据角色跳转对应首页
                                          → 登录失败 → 显示错误提示
```

### 5.2 招聘管理员流程
```
登录 → 首页（岗位列表）→ 选择操作
    ├─→ 新增岗位 → 填写表单 → 保存草稿/提交审批
    ├─→ 编辑岗位 → 修改信息 → 保存
    ├─→ 删除岗位 → 确认删除
    ├─→ 批量导入 → 上传Excel → 查看结果
    └─→ 提交审批 → DRAFT→PENDING
```

### 5.3 审批人流程
```
登录 → 首页（待审批列表）→ 选择岗位 → 审批页面
                                  ├─→ 通过 → PENDING→PUBLISHED
                                  └─→ 驳回 → PENDING→DRAFT
```

### 5.4 访客流程
```
访问首页 → 浏览岗位列表 → 点击查看详情 → 返回列表
```

## 6. API接口对接说明

### 6.1 岗位管理接口

| API路径 | HTTP方法 | 功能描述 | 招聘管理员 | 审批人 | 访客 |
|---------|---------|---------|-----------|-------|------|
| `/webapi/positions` | GET | 查询岗位列表 | ✓ | ✓ | ✓* |
| `/webapi/positions/{id}` | GET | 查询单个岗位 | ✓ | ✓ | ✓* |
| `/webapi/positions` | POST | 新增岗位 | ✓ | ✗ | ✗ |
| `/webapi/positions/{id}` | PUT | 修改岗位 | ✓ | ✗ | ✗ |
| `/webapi/positions/{id}` | DELETE | 删除岗位 | ✓ | ✗ | ✗ |
| `/webapi/positions/{id}/submit` | POST | 提交审批 | ✓ | ✗ | ✗ |
| `/webapi/positions/{id}/approve` | POST | 审批操作 | ✗ | ✓ | ✗ |
| `/webapi/positions/{id}/close` | POST | 关闭岗位 | ✓ | ✗ | ✗ |
| `/webapi/positions/upload` | POST | Excel上传 | ✓ | ✗ | ✗ |

* 访客仅能查看已发布状态的岗位

### 6.2 请求头要求

前端请求需在请求头中携带角色信息：
```
X-User-Role: RECRUITER  // 或 APPROVER / VISITOR
X-Authorization: Bearer {token}  // 登录后携带
```

## 7. 安全性要求

1. **登录验证**: 所有需登录的接口必须验证token有效性
2. **角色权限**: 后端根据请求头角色进行权限校验
3. **密码安全**: 密码传输使用HTTPS，存储使用加密算法
4. **防SQL注入**: 后端使用参数化查询
5. **XSS防护**: 前端对用户输入进行过滤和转义
6. **Session管理**: 支持token过期和刷新机制
